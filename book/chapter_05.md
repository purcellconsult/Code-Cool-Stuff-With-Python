<p>Tkinter supports 15 widgets. A widget is an object that provides various controls that the coder can use to integrate with the GUI.</p>
<p>A label as the name suggests allows you to display text or an image on the screen. Below is an example on how to create one:</p>
<pre><code>name = tk.Label(root, text='Your favorite food?').pack()

<p>So, we call the <code>Label</code> method using <code>tk</code>, and then pass in the root which is the parent of the class.</p>
<p>We can then make use of the various attributes in the label widget and text is one of them. As the name indicates it allows you to add text to the label.</p>
<p>It's important to note that the size of the label will automatically be set so that all of the characters in the text attribute are displayed. However, there‚Äôs an attribute that allows you to use the width and height of the label. Below is a listing of some of the attributes for the label widget.</p>
<p>‚ó¶ <code>width</code>: Change the width of the label widget. If you set this too small then all of the text won‚Äôt show.<br>
‚ó¶ <code>height</code>: Adjust the height of the label.<br>
‚ó¶ <code>bg</code>: The background color.<br>
‚ó¶ <code>fg</code>: The foreground color.<br>
‚ó¶ <code>bd/borderwidth</code>: The width of the label border.<br>
‚ó¶ <code>padx</code>: Extra horizontal padding to add around the text.<br>
‚ó¶ <code>pady</code>: Extra vertical padding to add around text.<br>
‚ó¶ <code>textvariable</code>: Associates a Tkinter variable, typically a StringVar with the label.</p>

<p>A <code>StringVar</code> is in essence part of the variable classes in tkinter; there's also the <code>BooleanVar</code>, <code>DoubleVar</code>, and <code>IntVar</code> classes. These serve as wrappers for their respective datatype.</p>

<h2><a id="user-content-pack" class="anchor" aria-hidden="true" href="#pack"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Pack</h2>

<h2><a id="user-content-place" class="anchor" aria-hidden="true" href="#place"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Place</h2>
<p>The place layout manager allows you to do absolute and relative positioning with tkinter. So, you can specify exactly where you want a widget to appear in the GUI. Here‚Äôs a code example that shows how to use the place geometry layout manager:</p>
<pre><code>import tkinter as tk

          'yellow', 'brown', 'gold']
width, height = 0, 0
for x in range(len(colors)):
    tk.Label(text=colors[x], width=10).place(x=0, y=height)
    tk.Label(bg=colors[x], width=15).place(x=100, y=height)
    height += 15
tk.mainloop()

<p>The last layout manager to explain is the grid. It organizes components of the GUI by placing them in a 2-dimensional table which consists of rows and columns. Widgets with the same row and column number will be placed on the same line. The size of the grid doesn‚Äôt have to be determined because the grid manager automatically takes care of this for you. The below code snippet utilizes the grid manager:</p>
<pre><code>import tkinter as tk

          'yellow', 'brown', 'gold']

for x in colors:
    tk.Label(text=x, width=15).grid(row=i+1, column=i)
    tk.Label(bg=x, width=15).grid(row=i, column=i)
    i += 1

<h2><a id="user-content-how-to-convert-procedural-gui-to-their-oop-counterpart" class="anchor" aria-hidden="true" href="#how-to-convert-procedural-gui-to-their-oop-counterpart"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>How to Convert Procedural GUI to Their OOP Counterpart</h2>
<p>The benefit of converting the procedural styled GUI into OOP is that in essence the program will be more modular and therefore makes debugging more precise. Let's take the very first GUI that we created in this chapter and then convert it to OOP style. To do this we need to accomplish 5 things:</p>
<ol>
<li>We need to create a class.</li>
<li>We need to create a constructor using the <strong>init</strong> method.</li>
<li>We need to prepend the self convention in front of the variables.</li>
<li>We need to create an instance of the class.</li>
<li>We need to call the mainloop method.</li>
</ol>
<p>Below is the Simple Tkinter GUI we created earlier in this chapter converted into OOP:</p>
<pre><code>import tkinter as tk
class HelloWorld:
    def __init__(self):
        self.root = tk.Tk()
